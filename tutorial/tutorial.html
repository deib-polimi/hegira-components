Simple Offline Data Migration Tutorial¶

In this tutorial we show how to perform a simple, offline data migration between two NoSQL databases.

Offline means that the system does not expect any write operation to occur in any of the databases from the moment when the migration is started till its conclusion. Doing otherwise would not guarantee data consistency between the two databases at the end of the process.

For the sake of simplicity, we will deploy all the necessary components in the same machine. Moreover, we suppose that the source database is already populated with some data.

In particular, this example will guide you to migrate your data from Google Datastore to Azure Tables.

Components configuration¶

In order to configure Hegira4Clouds, the databases credentials must be provided. Hence, a new file should be created under the folder ~/hegira-components/src/main/resources:

The file credentials.properties, for the sake of this tutorial, should contain the following entries:

azure.storageConnectionString=<escaped-azure-storage-connection-string>
datastore.username=<appengine-account-email-address>
datastore.password=<appengine-account-password>
datastore.server=<application-name>.appspot.com

After the credentials have been properly inserted, it is possible to build the remaining Hegira4Clouds components.

cd ~/hegira-components
mvn clean package -DskipTests
mv target/hegira-components-jar-with-dependencies.jar target/hegira-components.jar

Data migration execution¶

Once compiled two separate instances of the same application should be launched (either in background or in two separate shells).

One component should be started as follows:

java -jar target/hegira-components.jar -t SRC

While the other should be started as follows:

java -jar target/hegira-components.jar -t TWC

After the two components have been started, it suffices to execute an HTTP request to start the given data migration task:

curl -X POST -d "source=DATASTORE&destination=TABLES&threads=2" localhost:8080/hegira-api/api/switchover
